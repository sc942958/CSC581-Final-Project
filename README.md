# CSC581-Final-Project: Online Connect 4
<img width="1040" height="500" alt="Blank diagram(3)" src="https://github.com/user-attachments/assets/69699554-9359-4c59-9441-0c723789335b" />

My Project is a multiplayer online Connect 4 server. The application will allow 2 players to play a game of Connect 4 online, with the server being able to host more than 1 match at a time. The backend server will use the Nodejs base image. The application will also allow users to create an account, and store various statistics tied to that account such as win rate, games played etc.

The Project uses 1 dockerfile to build the nodejs container. The node container will have all of the frontend files, server application files, and database schemas and connection code. The node container will serve as the connection between the two users playing the game, and between each user and the database housing their information.

The first line builds from the latest version of the image since the project does not have version specific needs. The official nodejs image comes with the Node.js runtime preinstalled. Line 3 runs a pair of commands that creates the working directory for the node application (it is set as the working directory in line 5), and uses the "chown" command to enable node to edit files in the working directory for package management. Lines 7 to 13 are a mixture of "ADD" and "COPY" statements that copy the application files into the image. The "ADD" command is used to maintain the file structure within the image so the paths do not change. Line 15 sets the user to node, enabling line 3. Line 17 runs the "npm install" command, which installs the required pachages for the application, in this case Express, which adds additional server functionality, mongoose, which enables nodejs to connect to the mongodb, and EJS, a templating engine for frontend developement. Line 19 exposes the port 8080. Finally, line 21 executes the command to start "app.js", which brings the server online.

The second container in the project is the default mongodb image with no modifications, since the connection and schema code are part of the node container. The two containers will connect over the docker compose default private network using an exposed internal port on the mongo container with a url created for mongo.
